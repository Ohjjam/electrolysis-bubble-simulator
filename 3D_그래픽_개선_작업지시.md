# 3D 셀 그래픽 개선 작업지시

## 현재 상태 (2026-07-22)

아래 1~5번의 렌더링 구현은 현재 `web3d/app3d.html`과
`web3d/visuals3d.js`에 반영되어 있다. 물리 land mask와 분리된 bevel 형상,
실물 전체/내부/분석 프리셋, 원통 포트와 조립 디테일, 부착/이동 기포의
InstancedMesh 분리 변형, 저채도 반투명 기포와 림 하이라이트가 코드에 존재한다.

남은 완료 조건은 실제 브라우저에서 프리셋·레이어·임의 유로를 순회하는 시각 QA,
콘솔 오류 확인, 전후 FPS 측정이다. 이미 구현된 형상을 다시 작성하지 말고 이 검증과
발견된 회귀 수정부터 진행한다.

## 목표

현재 3D 패널의 물리 계산, 격자 좌표, 데이터 표시 기능은 유지하면서 셀·유로·포트·버블을 실제 전기화학 셀 조립체처럼 보이도록 개선한다.

단순히 폴리곤 수만 늘리지 말고 **계산용 복셀 형상과 화면용 렌더 형상을 분리**한다. 계산과 충돌 판정은 기존 격자/마스크를 그대로 사용하고, 시각 형상에만 라운딩·bevel·재질·조명 효과를 적용한다.

## 현재 확인된 문제

1. 기본 3D 자유 시점에서 불투명한 전면 엔드플레이트가 내부 구조 대부분을 가린다.
2. 전극면 전류맵의 강한 빨강–노랑–파랑 색이 형상과 깊이감을 덮는다.
3. 리브와 포트가 대부분 `BoxGeometry`라서 연속적인 가공 유로가 아니라 복셀 블록처럼 보인다.
4. 모든 버블이 완전한 구형이라 부착 기포와 이동 기포의 차이가 없다.
5. `EdgesGeometry`의 검은 외곽선이 실물보다 CAD/디버그 화면 느낌을 강하게 만든다.
6. 한쪽 전극 플레이트를 끄면 내부 구조가 훨씬 잘 보이므로 컷어웨이 구성이 적합하다.

## 절대 유지할 기능

- 음극 기포(H2) 레이어 토글
- 양극 기포(O2) 레이어 토글
- 음극 전극 레이어 토글
- 양극 전극 레이어 토글
- 막 레이어 토글
- 유로 벽(리브) 레이어 토글
- 전극면 전류맵 켜기/끄기
- 3D 자유, 정면 단면, 전극면, 위 시점
- 국소 확대 및 전체 보기 복귀
- 얇은 단면 보기와 3D clipping
- 실시간 버블 위치, 크기, H2/O2 구분
- 현재 물리 격자와 land mask, 포트 위치의 좌표 정합성
- 플레이트 에디터에서 변경한 임의 유로 형상
- 서버 API 및 Python 물리 모델

레이어 토글을 없애거나 기능을 단순화하지 않는다. 시각 프리셋을 추가하더라도 기존 개별 토글은 계속 동작해야 한다.

## 작업 전 주의사항

- 현재 작업 트리에 다른 수정사항이 많다. 기존 변경을 reset, checkout, stash, 덮어쓰기 하지 않는다.
- 먼저 현재 진행 중인 작업을 정상 동작 상태로 정리한 다음 그래픽 개선을 시작한다.
- 물리 계산 파일과 서버 API는 그래픽 개선을 이유로 변경하지 않는다.
- 가능하면 그래픽 관련 변경은 `web3d/app3d.html`과 별도 렌더 유틸리티 파일로 한정한다.
- 임의 유로 마스크와 버블 좌표가 어긋나지 않도록 계산 좌표계는 절대 변경하지 않는다.
- 외부 CDN 의존성을 새로 추가하지 않는다. 현재 vendored Three.js 환경 안에서 구현한다.

## 1. 컷어웨이 및 시각 프리셋

### 구현 목표

첫 화면에서 큰 엔드플레이트만 보이는 문제를 해결한다.

다음 프리셋을 제공한다.

1. **실물 전체**
   - 모든 셀 부품 표시
   - 전류맵 기본 OFF
   - 일반 3/4 시점
   - 실제 조립체 외관 확인용

2. **내부 보기**
   - 한쪽 엔드플레이트를 숨기거나 매우 낮은 투명도로 처리
   - 유로, 막, PTL, 버블이 명확히 보이는 3/4 컷어웨이 시점
   - 기본 진입 화면은 이 프리셋을 우선 검토

3. **분석 보기**
   - 전극면 또는 단면 직교 시점
   - 전류맵과 과학적 색상 표시 활성화
   - 현재의 정확한 복셀/외곽선 표현 사용 가능

### 조건

- 프리셋은 기존 레이어 토글을 호출해 조합하는 방식이어도 된다.
- 사용자가 프리셋 적용 후 개별 레이어를 다시 켜고 끌 수 있어야 한다.
- 국소 확대에서 전체 보기로 복귀했을 때 이전 프리셋과 레이어 상태가 복원되어야 한다.
- 컷어웨이 전용으로 부품을 이동시키는 경우 버블과 계산 좌표를 이동시키지 않는다. 실제 좌표를 왜곡하는 exploded view보다는 전면 부품 숨김/투명화를 우선한다.

## 2. 유로 리브의 bevel 및 부드러운 코너

### 구현 목표

계산용 land mask는 유지하면서 화면에서는 가공된 유로판처럼 보이게 한다.

### 권장 방식

- 리브 상단과 측면 모서리에 작은 bevel 또는 chamfer를 추가한다.
- bevel 크기는 실제 채널을 침범하지 않을 정도로 제한한다.
- 사행 유로의 U턴과 막힌 끝부분은 가능한 범위에서 둥근 끝단으로 표현한다.
- 임의로 그린 유로는 land mask로부터 외곽선을 추출해 렌더 형상을 생성하는 방식을 검토한다.
- 복잡한 마스크에서 연속 형상 생성이 불안정하면:
  - 정확한 복셀 형상은 분석 모드에 유지하고
  - 실물/내부 모드에서만 셀별 또는 병합 박스에 bevel을 적용한다.

### 금지

- 보기 좋게 만들기 위해 채널 폭이나 리브 위치를 임의 변경하지 않는다.
- 버블이 벽 안에 있거나 벽을 뚫고 보이게 만드는 좌표 불일치를 만들지 않는다.
- 매 프레임 geometry를 재생성하지 않는다. 마스크/격자 변경 시에만 재생성한다.

## 3. 실제 포트 및 셀 조립체 디테일

### 포트

- 현재 사각기둥 포트를 원통형 노즐 또는 튜브 피팅으로 표현한다.
- 포트가 위치한 면에 따라 CylinderGeometry 방향을 정확히 회전한다.
- 작은 플랜지 또는 연결부를 추가해 엔드플레이트와 자연스럽게 연결한다.
- 입구는 녹색, 출구는 적색을 유지하되 원색 발광을 줄이고 테두리 또는 링 색상으로 구분한다.
- 포트 중심과 폭은 기존 physics port mask에서 계산한다.

### 셀 조립체

과도하게 복잡한 CAD 모델을 만들지 말고 다음 시각 요소를 가볍게 추가한다.

- 엔드플레이트 모서리 라운딩 또는 chamfer
- 가스켓처럼 보이는 얇은 테두리
- 모서리의 볼트홀 또는 단순 체결 볼트
- PTL의 미세한 거칠기
- 막의 얇은 가장자리 강조

### 조건

- 볼트와 가스켓은 장식용이므로 물리 영역을 가리지 않게 배치한다.
- 작은 화면이나 분석 모드에서는 장식 형상을 숨길 수 있어야 한다.

## 4. 부착 버블과 이동 버블의 시각적 구분

### 현재 데이터 활용

버블 상태에는 side와 attached 정보가 있으므로 이를 렌더 변형에 사용한다. 새로운 물리 모델을 만들지 않는다.

### 부착 버블

- 완전한 구 대신 전극 법선 방향으로 눌린 spherical-cap 또는 타원체 형태로 표시한다.
- 음극/양극 면의 방향에 맞게 변형 축을 설정한다.
- 전극과 맞닿은 부분이 떠 보이지 않도록 위치와 스케일을 함께 보정한다.
- 접촉각을 시각 변형에 반영할 수 있으면 기존 `theta` 값을 사용하되 물리 위치를 변경하지 않는다.

### 이동 버블

- 기본 구형을 유지하되 이동 방향으로 아주 약한 타원 변형을 허용한다.
- 과장된 젤리 애니메이션이나 불규칙한 흔들림은 넣지 않는다.

### 재질

- 현재처럼 H2/O2 전체를 강한 원색으로 칠하기보다 투명한 본체와 얇은 색상 테두리로 구분하는 방향을 우선한다.
- Fresnel 또는 림 하이라이트를 이용해 밝은 배경에서도 경계를 읽을 수 있게 한다.
- `transmission` 등 고비용 효과를 모든 인스턴스에 적용해 성능을 크게 떨어뜨리지 않는다.
- 필요하면 가까운 큰 버블과 원거리 작은 버블을 서로 다른 렌더 품질로 나눈다.

### 성능

- 현재 수천 개의 버블이 표시될 수 있으므로 InstancedMesh 기반을 유지한다.
- 버블마다 별도 Mesh를 생성하지 않는다.
- attached/free를 두 개의 InstancedMesh로 나누는 것은 허용하되 전체 draw call과 CPU 갱신 비용을 확인한다.

## 5. 조명, 깊이감 및 재질

### 조명

- 현재 RoomEnvironment와 PBR 재질을 기반으로 개선한다.
- 부드러운 key light와 반대편 rim light로 셀 층 구조가 구분되게 한다.
- 과한 bloom이나 영화식 효과는 피한다.
- 흰 배경에서 투명 부품이 사라지지 않도록 배경을 매우 옅은 실험실 톤의 그라데이션으로 조정한다.

### 깊이감

- 실물/내부 모드에서는 검은 `EdgesGeometry` 외곽선을 약화하거나 숨긴다.
- 분석 모드에서는 정확한 격자 확인을 위해 외곽선을 유지할 수 있다.
- bevel 하이라이트, 재질 roughness 차이, 약한 AO 느낌으로 부품 경계를 표현한다.
- 실제 SSAO 후처리가 너무 무겁거나 현재 번들에 없으면 무리하게 추가하지 말고, 조명과 재질로 유사한 깊이감을 만든다.

### 재질 가이드

- 엔드플레이트: 어두운 금속/코팅 소재지만 완전한 검정은 피한다.
- 리브/유로판: 음극과 양극 계열 차이를 유지하되 채도를 낮춘 금속 또는 엔지니어링 플라스틱 느낌.
- PTL: 반투명 유리판이 아니라 거칠고 다공성인 층처럼 표현.
- 막: 매우 얇고 밝은 반투명 필름.
- 전류맵: 기본 실물 모드에서는 OFF, 분석 모드에서는 현재 기능 유지.

- 셀 크기, 채널 수, 복셀 해상도 변경 후 좌표 정합성

### 그래픽 확인

- 기본 화면에서 내부 구조가 바로 읽혀야 한다.
- 리브가 평면 줄무늬가 아니라 두께가 있는 유로 벽으로 보여야 한다.
- 포트가 셀에 연결된 피팅처럼 보여야 한다.
- 부착 기포가 전극에 닿은 형태로 보여야 한다.
- 버블이 리브 밖으로 튀어나오거나 벽을 관통해 보이지 않아야 한다.
- 전류맵을 껐을 때도 셀 구조와 버블이 충분히 구분되어야 한다.
- 분석 모드에서는 기존 데이터 해석성이 유지되어야 한다.

### 성능 및 오류

- 브라우저 콘솔에 새 오류가 없어야 한다.
- 기존 렌더 FPS가 눈에 띄게 악화되지 않아야 한다.
- 현재 `window.__rinfo()` 등의 진단 함수를 유지한다.
- geometry/material이 반복 생성되며 누적되지 않는지 확인한다.
- 서버와 Python 테스트가 그래픽 변경으로 깨지지 않아야 한다.

### 결과물

- 변경 파일 목록
- 구현 내용 요약
- 각 프리셋과 국소 확대 스크린샷
- 레이어 토글 및 유로 변경 테스트 결과
- 렌더링 성능 전후 비교
- 실행한 테스트와 결과

## 최종 원칙

이 작업은 시뮬레이터를 게임처럼 화려하게 만드는 작업이 아니다. **실제 실험 장비처럼 신뢰감 있게 보이면서 내부 물리 현상을 이해하기 쉬운 과학 시각화**를 만드는 것이 목표다.
