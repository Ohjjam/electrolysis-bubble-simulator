# 3세대(3D) 클라우드 배포 가이드 — Hetzner VPS

> 목표: 월 ~₩6천짜리 서버 한 대에 3세대 시뮬을 올려서, 누구나 **링크만으로** 접속.
> 님은 코드 고치고 `git push`(또는 `클라우드 배포.bat`)만 하면 서버가 알아서 최신으로 갱신.
> 리눅스 명령어는 **한 줄도 안 쳐도 됨** (서버 생성 때 스크립트 한 번 붙여넣기만).

---

## 0. 지금 가입 화면(계정 확인)이 떴다면

Hetzner 신규계정 표준 절차예요(사기 아님). **Credit card**(빠름, 보통 즉시~몇 분) 또는
**Document**(여권/신분증, 사람이 검토 → 수 시간). → **Credit card 추천.** 카드 정보는
**본인이 직접** 입력(제가 대신 못 함). 통과돼야 서버 생성 버튼이 열려요.

## 1. 서버 생성 (Hetzner Cloud 콘솔 → "Add Server")

| 항목 | 고를 것 |
|---|---|
| **Location** | `Hillsboro (US West)` — 한국에서 제일 가까움 |
| **Image** | `Ubuntu 24.04` |
| **Type** | `CPX11` (AMD, 2 vCPU / 2GB, ~₩6천/월). *코어 많은 거 사도 이 앱은 안 빨라짐* |
| **SSH key / Password** | 아무거나 (안 써도 됨). 나중을 위해 root password 하나 정해두면 편함 |
| **Cloud config** ⭐ | 아래 2번 스크립트를 **여기에 붙여넣기** |
| 나머지 | 기본값 그대로, **Create & Buy now** |

## 2. "Cloud config" 칸에 붙여넣을 것

`deploy/cloud-init.yaml` 파일 내용을 **통째로 복사**해서 붙여넣으면 됩니다.
(서버가 첫 부팅에 알아서: git clone → python·numpy 설치 → 앱 실행 → HTTPS 발급)

## 3. 3~5분 기다린 뒤 접속

주소는 **서버 IP 기반**으로 자동 생성돼요:
`https://<서버IP를 점 대신 하이픈으로>.sslip.io`
예) IP가 `5.161.12.34` → **https://5-161-12-34.sslip.io**

- 서버 IP는 Hetzner 콘솔의 서버 카드에 나와요.
- 첫 접속은 인증서 발급으로 10~30초 걸릴 수 있음(이후 즉시).
- 확인용 로그: 콘솔의 서버 → `Console`(웹 터미널)에서
  `cat /var/log/bubblesim-setup.log`, `cat /opt/bubblesim/deploy/PUBLIC_HOST.txt`

---

## 4. 앞으로 업데이트하는 법 (이게 핵심 편의)

코드 고친 뒤 **둘 중 하나**:
- 평소 쓰던 대로 `git push origin main`, 또는
- 루트의 **`클라우드 배포.bat` 더블클릭** (전체 변경 커밋 + 푸시)

→ 서버가 **2분 안에 자동으로** 최신 코드로 갱신 + 재시작. 서버 로그인 불필요.
(구조: 서버에 2분마다 `git pull` 하는 cron이 `deploy/update.sh` 실행)

> ⚠️ `클라우드 배포.bat`은 **현재 폴더의 모든 변경**을 커밋해서 올립니다.
> 세밀하게 나눠 커밋하고 싶으면 그냥 평소 git 명령을 쓰세요.

## 5. 알아둘 것

- **속도**: 실측 결과 이 시뮬은 코어 1개 성능만 중요(멀티코어 효과 없음). 클라우드에서도
  로컬 PC와 비슷한 ~0.1~0.2배속(원래 이 정도)입니다. CPX11로 충분.
- **자동 일시정지**: 아무도 안 보면(45초간 접속 없음) 계산을 멈춰 CPU를 놓아줍니다.
  누가 접속하면 즉시 재개. → 나중에 이 서버에 **다른 앱(게임 등)도 같이** 올렸을 때
  놀고 있는 시뮬이 CPU를 잡아먹지 않음.
- **동시 접속**: 지금 구조는 접속자 전원이 **시뮬 1개를 공유**합니다. 한 명씩 볼 땐 완벽,
  여러 명이 동시에 만지면 화면·설정이 섞여요. (각자 따로 돌리려면 별도 작업 필요 — 추후)
- **비용**: 서버는 정액(월 ~₩6천). 앱을 여러 개 올려도 값은 그대로. RAM/CPU 부족하면
  Hetzner 콘솔에서 클릭 한 번으로 상위 타입으로 **Resize**(재설치 없음, 몇 분 다운타임).
- **끄기/삭제**: 콘솔에서 서버를 Power off 하면 과금은 계속(디스크 점유). 완전히 안 쓰면
  **Delete** 해야 과금 종료.

## 6. 이 서버에 다른 프로젝트도 올리려면 (나중에)

한 대에 여러 앱을 올릴 수 있어요. 각 앱을 다른 내부 포트로 띄우고, Caddy에 항목만 추가:
```
# /etc/caddy/Caddyfile 예시
5-161-12-34.sslip.io        { reverse_proxy 127.0.0.1:8766 }   # 버블시뮬
game.5-161-12-34.sslip.io   { reverse_proxy 127.0.0.1:3000 }   # 멀티플레이 게임
```
→ 그때 말해주면 이 서버 기준으로 세팅 잡아줄게요.

## 7. 문제 생기면 (웹 콘솔에서)

```bash
systemctl status bubblesim3d          # 앱 상태
journalctl -u bubblesim3d -n 50        # 앱 로그
systemctl status caddy                 # HTTPS 상태
cat /var/log/bubblesim-setup.log       # 최초 설치 로그
cat /var/log/bubblesim-update.log      # 자동 업데이트 로그
sudo bash /opt/bubblesim/deploy/setup.sh   # 설치 다시(멱등)
```
그래도 안 되면 위 로그 복사해서 나한테 주면 됨.
