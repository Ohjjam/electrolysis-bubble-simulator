# 3-D 전기화학 셀 시뮬레이터 (`bubblesim3d`)

기존 평면(0D~2D) 시뮬레이터(`bubblesim/`, `app.html`, `server_app.py`)는 그대로 두고,
**현실의 3D 구조**(3D 전극·유로·다공 스캐폴드)를 다루는 별도 시뮬레이터를 신설한 것.
검증된 `bubblesim.kernel` 물리(Butler–Volmer, 물성, 화학, Faraday 소스, 이탈력)를
공통으로 재사용한다. 새 항목은 컨텍스트에 add-only로 추가하고 기존 키 계약은 유지한다.

## 두 트랙

### Track A — 셀 스케일 3D (라이브, 인터랙티브)
- **격자**: mm급 균일 복셀. 전극면(x), 유동/높이(y, 부력), 셀 폭(z).
- **유동**: staggered MAC 격자 위 비압축 Navier–Stokes (Chorin projection +
  semi-Lagrangian 이류 + red-black SOR 압력 풀이, warm-start). 발산은 기계 정밀도까지 제거.
  명시적 점성항은 없음(Stam 방식 — 수치확산+damping); 정량 경계층이 아니라 부력 플룸·
  self-stirring·기울기 가스 갇힘을 보여주는 정성 시각화.
- **기포**: Lagrangian 가스 파슬 — 전극면에서 Faradaic 율(`kernel.sources`)로 핵생성,
  볼륨 보존, Schiller–Naumann 종단속도(`kernel.population`와 동일식)로 상승, 3×3×3 보이드
  투영 → Boussinesq 부력(양방향 결합). 총 가스량 = ∫j/nF 보존.
- **대표 다중도**: 추적 기포 1개 = 실제 기포 `mult`개. 반경 `r`은 진짜 단일 기포 반경,
  가스량은 `W = mult·V(r)`. 성장·이탈·솎아내기 어디서도 `W = mult·V(r)`와
  `생성 = 잔류 + 배출`이 기계 정밀도로 유지됨.
- **벽 구속(confinement)**: 반대편 플레이트가 `d_ch` 떨어져 있으므로 어떤 기포도
  `0.45·d_ch`를 넘을 수 없음. 이탈 반경과 성장 모두 이 한계로 clamp — 넘치는 가스는
  더 큰 기포가 아니라 **같은 크기의 기포 더 많이**(`mult` 증가)가 됨.
- **성장 부스텝**: 실제 사이트 밀도에서 씨앗→이탈 사이클은 ~1 ms로 유동 시간스텝(3 ms)보다
  짧다. 한 번에 다 성장시키면 기포가 이탈 반경의 3배까지 부풀어 오르므로, 벽 배열만
  사이클의 25% 이하씩 부스텝(`Parcels.SUB_FRAC`). 유동 풀이는 부스텝하지 않음.
- **합체(coalescence)**: 액체 속 기포는 "터지지" 않고 **닿으면 합쳐진다**(터지는 건 기액
  분리기 자유표면 = 여기선 outlet vent). 이웃은 자기 자신의 `mult` 형제들이라 추적
  해상도 아래에 있으므로, 성장으로 늘어난 접촉 확률의 **증가분** `dP`에 대해 병합을
  추첨(스텝 크기에 무관). salting-out은 `η=c_crit/(c_crit+c)` 연속 효율로 합체를 억제하며, 고농도에서도
  합체확률을 임의로 0으로 만들지 않는다.
  병합 시 `mult`가 절반, `r`이 2^(1/3)배 → `W` 불변.
  상승 중인 자유 기포군은 기하 충돌 커널×Prince–Blanch 액막 배수 효율로 합체한다.
- **PP mesh 수집**: 카탈로그의 두 개구 치수·개방률·두께로 실제 y/z strand 좌표를 만들고,
  양극 기포가 원통 strand와 기하학적으로 닿을 때만 NF→PP로 전달한다. mesh 부착 상태는
  전극 부착/자유 기포와 분리하며, 접촉한 두 대표 cohort만 가스체적 보존형으로 합체한다.
  PP 위 기포는 strand 방향으로 이동하고 끝단에서 방출되거나, Schiller–Naumann 유동 항력+부력이
  정적 Young–Dupré 부착력 추정을 넘을 때 자유 기포가 된다. 전진·후진각은 입력하지 않으므로
  접촉각 이력/접촉선 pinning은 계산하지 않는다.
- **전기화학**: 스칼라 작동점(셀 전압, 평균 j, 과전압)은 `ZeroDTwoElectrodeSolver`가
  N스텝마다 재계산(CachedSolver식). 3D 엔진은 그 위의 필드 시각화.
- **입구 / 출구 포트**: 입구는 y=0 면의 지정 법선속도, 출구는 y=Ly 면의 Dirichlet p=0.
  둘 다 **폭·위치를 고를 수 있는 포트**이며, 출구를 좁히면 윗면의 나머지는 **플레이트**가 됩니다
  — 액체뿐 아니라 **기체도** 그 밑에 갇혀 포트를 찾아 옆으로 기어갑니다. 출구를 왼쪽/오른쪽에
  두면 상단 가스 분포 중심이 실제로 이동합니다 (측정: z-중심 0.45 ↔ 0.28).
- **유로 직접 그리기 (`ff="custom"`)**: 32×32 마스크를 브라우저에서 칠하면 그대로 `land_mask`가
  되어 격자에 리샘플됩니다(전극 크기·복셀 수를 바꿔도 그림은 보존). 전부 리브로 칠하면 채널
  하나는 강제로 열어 둡니다 — 유체 셀이 없으면 풀이가 성립하지 않기 때문.
  주의: 그린 판을 `in_w=0`(바닥 전체 급수)으로 돌리면 남긴 틈으로 유량이 몰려 제트가 생깁니다
  (측정 2.9 m/s vs 펌프 0.35 m/s). 실제 판처럼 포트를 지정하세요.
- **유로 복셀화**: 디자이너의 유로(사행/병렬/교차)를 리브 장애물 마스크로 만들어 유동이
  리브를 우회 → 전극면 전류 재분배 `j(y,z)=j_mean·(1-θ)/mean(1-θ)`(전하 보존, 실제 3D
  파슬 기반 θ). "리브 섀도잉 → 불균일 전류"를 컬러맵으로.

### Track B — 포어 스케일 3D (오프라인 배치 → 재생)
- **미세구조**: `kernel.morphology`의 기재별 목표(공극률, 비표면적, R_f)로 복셀 스캐폴드 생성.
  leveled-wave Gaussian random field(numpy.fft) → 이중연속(양 상 관통) 구조. 공극률 정확히
  일치(폼은 64³ 해상도 한계로 연결성 위해 하한 적용, `eps_achieved`로 명시; 128³에서 회복).
- **전류분포**: 접근면(y=0)에서 처음 만나는 외부 `pore|solid` 계면만 반응면으로 둔다.
  열린 외부 사이트에 총전류를 균일 분배하고, 가스로 막힌 사이트는 제외한다. 다공전극
  침투길이·screened-Poisson·내부 BV 반응은 사용하지 않는다.
- **기포 성장(VOF 아님)**: 외부 반응면에서 Faradaic 가스를 생성, 연결된 열린 포어를 복셀 단위로
  채움(핵생성=고전류·고사이트 우선, 성장=기존 가스 팽창), 분리막면에 도달한 클러스터는
  배출(escape_factor 반영). **가스 보존**: produced = resident + vented (복셀 1개 이내).
- **실행**: `python -m bubblesim3d.runner --substrate ni_foam --grid 64 --frames 60`
  또는 앱 재생 탭의 "배치 계산 시작" 버튼. 결과는 `results/<run>/`에 프레임별 저장
  (가스 마스크 packbits, 용존장 float16 → 64³ run이 수십 MB). 타임라인 스크러버로 재생.

## 앱 / 서버
- **`web3d/app3d.html`** (+ 벤더링 three.js `web3d/vendor/`): Live(A)/Playback(B) 모드 스위치.
  Live: InstancedMesh 기포 + 전류밀도 DataTexture. Playback: 스캐폴드/가스 복셀을 instanced
  cube로 렌더 + 스크러버.
- **`web3d/app2d.html`** (`/2d`): 같은 계산의 app.html식 2패널 평면 뷰 — ① 촉매면 µm 확대,
  ② 전체 셀 가스 막힘 지도.
- **유로 템플릿**: 사행 · 세로 사행 · 병렬 · 지그재그 · 핀(엇갈림) · 핀(정렬) · 나선 · 빈 판.
  현재 격자에서 바로 생성되므로 정확하고 곧바로 수정 가능. 채널 수·리브 두께는 격자가 담을 수
  있는 값으로 clamp된다(2016개 조합 전수 검사: 전부 입구→출구 연결). 교차형은 이 모델에서
  유동 경로가 없어 제외.
- **`web3d/plate_editor.js`**: 두 뷰가 공유하는 유로 에디터. **물리 격자 해상도로 그리므로
  한 칸이 솔버 셀 하나**입니다(WYSIWYG). 도구: 직선(축 자동정렬)·사각형·붓·지우개, 되돌리기,
  좌우대칭, 크게 보기. 한 획은 격자 재생성 **1회**로 모읍니다. 마스크는 `ny,nz:0101…` 형태로
  모양을 함께 싣고, 입·출구는 **엔진이 실제로 적용하는 포트**(`state.ports`)를 그립니다.
- **`web3d/controls.js`**: 두 뷰가 **공유하는 단일 컨트롤 정의**. 한쪽에서 값을 바꾸면
  서버(`/api3d/op`)가 유일한 진실이 되고, 다른 뷰는 다음 폴링(~110 ms)에 `sync()`로 위젯을
  맞춥니다. 드래그 중인 위젯은 덮어쓰지 않습니다. 두 화면의 수치가 어긋날 수 없습니다.
- **그려진 셀 = 계산 격자**: 3D 뷰의 판·리브·기포 좌표는 전부 `st.grid`(nx,ny,nz,h)에서
  나옵니다. 디자이너의 `W_cm/H_cm`로 판을 그리면 복셀 반올림(과 `max_cells` 캡) 때문에
  `nz·h ≠ W_cm·10`이 되어 리브가 기포에 대해 미끄러지고, 기포가 유로 밖으로 튀어나와
  보였습니다 (7.5 cm 셀에서 1 mm = 리브 반쪽).
- **`server3d_app.py`** (:8766, `server_app.py` :8765와 분리): `LiveSim3D`(Track A 데몬 스텝)
  + Track B 재생 파일 서버 + 배치 잡 매니저. `/api3d/state`·`/api3d/op`·`/api3d/runs`·
  `/api3d/scaffold`·`/api3d/frame`·`/api3d/run`. 분리 이유: 배포된 라이브 앱과 Pyodide
  파이프라인 무영향, Track B는 파일시스템·장기 작업 필요.
- 실행: `run3d.bat` (Windows, Python 3.14).

## Honest scope (모듈 docstring에 명시)
- Track A: 명시적 점성 없는 Stam 유동(정성 시각화), 면 전류는 전극-네트워크 축약(3D 갭
  퍼텐셜 풀이 아님).
- **기포의 벽면-법선 이동**: Tomiyama(2002) 전단 양력 + Antal(1991) 벽 윤활력을 Stokes
  항력과 균형시켜 푼다. 전기분해 크기(Eo_d ≈ 1e-3)에서는 양력이 모든 이격거리에서 이겨
  기포를 전극에 눌러붙이므로 `y_w ≈ 1.05 r`에 앉는다 — 근벽 기포층은 이제 **힘균형의 결과**다.
  (Eo_d > 4에서는 C_L 부호가 뒤집혀 mm 기포가 중심부로 이동.)
- **자유 무리 합체**: Prince–Blanch(1990) 충돌 커널 × 필름 배수 효율 × 전해질 억제.
  관측량은 **출구 기포 크기**(6 M KOH ≈ 42 µm, 0.1 M ≈ 84 µm). 닫힘식 1개(셀 평균 보이드율을
  근벽 시트로 농축, ε_max = 0.62에서 절단)를 쓰며, 두 경계 모두 답을 지배하지 않는다.
- **난류 분산 없음**: 이 셀은 층류(Re ≈ 4e2)라 Prince–Blanch 난류항이 적용되지 않는다.
- **mesh 유동 해상도**: strand는 기포 수집 표면으로는 3D에 존재하지만 woven mesh 주위 유동을
  복셀 CFD로 직접 해상하지 않는다. Mesh 1의 단면감소/압력손실은 1D LSV 축약식, Mesh 2는 그
  수력학 효과를 끈 분리 실험이다.
- Track B: 전기화학 반응은 외부 접근 표면에만 있는 축약모델이다. 내부 포어는 가스가 이동·점유할
  수 있는 기하일 뿐 반응 침투를 나타내지 않는다. 기포는 복셀 채움(계면 추적 VOF 아님),
  다공구조는 통계적 대표 구조(특정 전극의 토모그래피 재구성 아님)다.
